# ADR-005: Why ChromaDB for RAG

**Date**: 2026-05-14  
**Status**: Accepted  
**Deciders**: Solo founder

---

## Context

Agents 7 and 9 need access to a library of proven HTML/CSS patterns for small business websites. The system must retrieve relevant patterns based on semantic similarity (business type + section) with low latency and zero cloud cost.

## Decision

Use **ChromaDB** as the vector store for the RAG pattern library, running locally on the same server as FastAPI.

## Alternatives Considered

| Option | Rejected Because |
|--------|-----------------|
| **Pinecone** | Managed cloud service — costs money at scale; adds network latency; external dependency |
| **Weaviate** | Overkill for 60 documents; complex setup; self-hosted or paid cloud |
| **pgvector** | Would require Postgres extension; adds complexity to Supabase; RAG queries would compete with transaction queries |
| **FAISS** | No built-in persistence or metadata filtering; requires more glue code |
| **Simple string matching** | No semantic similarity — would miss relevant patterns for uncommon business types |

## Reasoning

**Zero cost**: ChromaDB is open-source and runs in-process with FastAPI. No cloud service fees.

**Persistent local storage**: ChromaDB persists to filesystem (`CHROMA_PERSIST_PATH=./chroma_db`). Data survives FastAPI restarts. No external DB needed.

**Metadata filtering**: ChromaDB supports filtered queries (`where={"category": "restaurant"}`). This lets agents narrow pattern retrieval by business type before semantic search.

**Python-first**: ChromaDB has a clean Python client that integrates naturally with FastAPI. No additional language barrier.

**Scale is appropriate**: The RAG collection has ~60 patterns at launch. ChromaDB handles millions of documents — this is well within its comfort zone.

**Embedding model included**: ChromaDB works with sentence-transformers (all-MiniLM-L6-v2) locally, so no OpenAI embedding API calls needed.

## Trade-offs Accepted

- Local filesystem storage — no replication (acceptable at v1 scale)
- ChromaDB process runs inside FastAPI — not a separate service (simpler but less isolated)
- If `chroma_db/` directory is lost, must re-seed from `raw/10_rag_seed_content.md`

## Consequences

- RAG patterns seeded from `raw/10_rag_seed_content.md` at first startup
- Pattern retrieval adds ~50ms to Agent 7 and Agent 9 calls (local disk access)
- Collection backed up as part of server filesystem backup
- New patterns added via Python client or FastAPI admin endpoint
