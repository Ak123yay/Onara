# ADR-001 — FastAPI Over Serverless for the Pipeline

**Status**: Accepted | **Date**: 2026-05-14 | **Source**: raw/Onara_Business_Plan_v12.md

---

## Context

The 10-agent pipeline requires 60–120 seconds of continuous execution. Serverless functions (Vercel, Supabase Edge, AWS Lambda) have maximum execution times of 10–60 seconds — architecturally incompatible.

---

## Decision

Use FastAPI on a persistent server (PC + Cloudflare Tunnel at launch; DigitalOcean Droplet in production) for all AI pipeline work.

Next.js on Vercel handles only: auth validation, plan limit enforcement, forwarding to FastAPI, and receiving pipeline results.

---

## Rationale

1. **No execution timeout**: persistent server, no platform cap. Pipeline runs 2+ minutes freely.
2. **Local model access**: Ollama runs on the same machine — zero latency, zero cost for local models.
3. **Simpler pipeline control**: plain Python + Blackboard = easier to debug than chained serverless functions.
4. **No LangChain/LangGraph**: useful when you don't control infrastructure. Onara controls everything — they add complexity with no benefit.
5. **Cost**: PC at launch = $0. DigitalOcean = $12–24/month. Breakeven: 2 paying Starter users.

---

## Consequences

- PM2 auto-restarts FastAPI on crash. UptimeRobot monitors `/health`.
- Dev: Cloudflare Tunnel provides persistent public URL for local server.
- v1.5: Add BullMQ + Redis when average queue wait > 5 minutes.

---

## Rejected Alternatives

| Alternative | Reason |
|-------------|--------|
| Vercel Functions | 60s execution limit |
| Supabase Edge Functions | 400ms wall clock limit |
| AWS Lambda | Same timeout problem; added AWS complexity |
| LangChain/LangGraph | Extra dependency, harder to debug, no benefit |
