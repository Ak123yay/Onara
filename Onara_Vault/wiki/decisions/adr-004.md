# ADR-004: Why Stripe

**Date**: 2026-05-14  
**Status**: Accepted  
**Deciders**: Solo founder

---

## Context

Onara needs recurring subscription billing (monthly + annual), a self-serve upgrade/cancel flow, webhook-based lifecycle management, and a hosted payment page (no custom card UI).

## Decision

Use **Stripe** for all payment processing and subscription management.

## Alternatives Considered

| Option | Rejected Because |
|--------|-----------------|
| **Paddle** | Merchant of Record (good for international tax) but more complex setup; smaller ecosystem; less documentation |
| **LemonSqueezy** | Similar to Paddle; designed for digital products; less mature API |
| **Braintree** | Lower-level API; requires more custom UI work; PayPal-owned (trust/complexity) |
| **Manual invoicing** | Not scalable; no automated trial lifecycle; no webhooks |
| **Supabase + custom billing** | Would require building subscription logic from scratch |

## Reasoning

**Stripe Checkout**: Hosted payment page means zero PCI scope for Onara. Users enter card details on Stripe's servers — Onara never sees card data.

**Customer Portal**: Stripe's hosted portal handles upgrades, downgrades, cancellations, and invoice downloads. Zero custom UI to build or maintain.

**Webhook reliability**: Stripe webhooks are the most mature event system in the industry. The `STRIPE_WEBHOOK_SECRET` HMAC verification pattern is well-documented and battle-tested.

**Developer experience**: `stripe` npm package is the gold standard. Error handling, retry logic, and type safety are all built-in.

**Trial period support**: Stripe natively supports trial periods in checkout sessions — but Onara uses a custom reverse trial (14-day full access, no card upfront) which requires custom subscription creation, not Stripe's trial parameter. Stripe handles both patterns.

**Pricing**: 2.9% + $0.30 per transaction — competitive; no monthly fees until transaction volume justifies alternatives.

## Trade-offs Accepted

- 2.9% + $0.30 per transaction (Paddle/LemonSqueezy may handle international tax better)
- US/EU focus — international tax compliance is handled by Stripe Tax (add-on, future)
- Stripe is a critical dependency — if Stripe is down, billing is down

## Consequences

- All billing state in Stripe (subscription IDs, customer IDs) + mirrored in Supabase `subscriptions` table
- Webhook endpoint at `/api/billing/webhook` handles all subscription lifecycle
- Customer Portal at `/api/billing/create-portal-session` → Stripe-hosted URL
- 6 environment variables required (see `wiki/architecture/env-vars.md`)
