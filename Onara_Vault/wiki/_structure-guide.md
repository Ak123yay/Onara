# Wiki Structure Guide

## Purpose
This folder is a structured knowledge base managed by Claude. Multiple files can be created.

---

## Folder meanings

### architecture/
System design, modules, and overall structure of the project.

### features/
Individual product features and requirements.

### ai-agents/
Logic, workflows, and behavior of AI systems.

### data/
Schemas, models, and data structures.

### decisions/
Architecture decisions and tradeoffs (ADR format).

### research/
Raw analysis, notes, and external findings.

### testing/
Test strategy and validation logic.

---

## Rules
- Every folder must have an _index.md
- Every file must belong to a category
- Cross-link using [[wiki links]]
- Keep content concise and structured
