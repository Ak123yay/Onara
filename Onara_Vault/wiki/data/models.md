# Data Models — Supabase Schema

_Complete PostgreSQL schema, RLS policies, triggers, and pg_cron jobs. Source: raw/02_database_schema.md._

---

## Overview

Onara uses Supabase (hosted PostgreSQL). RLS enabled on every table. Pipeline server uses service role key; frontend uses anon key (RLS enforces access boundaries).

---

## Custom Types

```sql
plan_type:     'free' | 'starter' | 'pro'
site_status:   'queued' | 'generating' | 'deploying' | 'live' | 'failed' | 'suspended'
agent_phase:   'analyst' | 'content_writer' | 'style_agent' | 'planner' |
               'prompt_engineer' | 'code_generator' | 'debugger' | 'seo_agent' |
               'qa_agent' | 'mobile_agent' | 'deploying' | 'done' | 'error'
revision_type: 'initial_generation' | 'user_revision' | 'system_refresh'
```

## Row Level Security

| Table | Policies |
|-------|----------|
| `users` | SELECT + UPDATE own row only |
| `projects` | Full CRUD own projects only |
| `pipeline_jobs` | SELECT own jobs only (pipeline writes via service role) |
| `revisions` | SELECT + INSERT own only |
| `pipeline_errors` | SELECT own job errors only |
| `gbp_sync_log` | SELECT via project ownership check |
| `stripe_events` | No user access (`USING (FALSE)`) |

---

## Triggers

| Trigger | Action |
|---------|--------|
| `users_updated_at` | Auto-updates `updated_at` on users |
| `projects_updated_at` | Auto-updates `updated_at` on projects |
| `on_auth_user_created` | Creates `public.users` row on `auth.users` INSERT |
| `check_project_limit` | Enforces plan project caps on INSERT to projects |
| `on_revision_created` | Increments `users.revisions_used` on revision INSERT |

## pg_cron Jobs

| Job | Schedule | Action |
|-----|----------|--------|
| `reset-free-revisions` | `0 0 1 * *` (1st of month, 00:00 UTC) | Resets `revisions_used = 0` for all free users |
| `downgrade-expired-trials` | `0 2 * * *` (daily 02:00 UTC) | Downgrades users where `is_trial=TRUE AND trial_ends_at < NOW()` |
| `suspend-failed-payment-sites` | `0 3 * * *` (daily 03:00 UTC) | Sets `status='suspended'` for `past_due` subscribers |
| `cleanup-pipeline-errors` | `0 4 * * 0` (Sunday 04:00 UTC) | Deletes errors older than 30 days |
