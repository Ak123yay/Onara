# Contractor-Specific Features — Onara Generated Sites

_Components and sections that Onara generates specifically for home-service contractors: plumbers, electricians, HVAC, handymen, roofers, landscapers, and similar trades._

_These are passed as structured data through the Blackboard to Agent 3 (Style) and Agent 6 (Code Generator). Agent 1 (Analyst) determines which apply based on the business category from GBP._

---

## Why Contractors Are Different

A contractor site has different conversion goals than a restaurant or salon:

- **Primary action**: Phone call (not a booking form or menu)
- **Primary fear**: "Will they show up? Are they legit?"
- **Primary hook**: Availability + proximity ("We're in your area, we're available now")
- **Trust signals**: License, insurance, years in business — not photos or ambiance

Every feature below maps to one of those four goals.

## Feature 1 — Emergency Banner

**Priority**: v1  
**Applies to**: Plumbers, electricians, HVAC, roofers, locksmiths  
**Blackboard field**: `has_emergency_service` (bool, from Agent 1 classification)

### What it is

A full-width sticky banner above the nav in a high-contrast color (red, orange, or dark navy depending on palette). Contains:
- Short emergency hook ("Burst pipe? Frozen pipes? Water heater out?")
- Prominent phone number
- "24/7" or "Emergency service available" label

### Design reference

See `PlumberMock` in `Onara_Design/p-sitemocks.jsx` — the orange phone block "Burst pipe? We're 20 min out." with large phone number and "24 / 7" tag.

### Implementation notes

- Agent 3 selects the palette: utility bold (navy + safety orange) for plumbers/HVAC, dark/industrial for electricians
- Agent 6 renders the banner above `<header>` as a `position: sticky; top: 0` strip
- Phone number pulled from `business_phone` on the Blackboard
- Emergency copy generated by Agent 2 (Content Writer) using the `emergency_hook` prompt slot

### Agent 2 prompt slot — emergency_hook

> "Write a 6–10 word emergency headline for a [category] business. Use the customer's pain (broken, leaking, no heat) not the service name. No filler words."

Example outputs:
- Plumber: "Burst pipe? Flooding? We're 20 minutes out."
- HVAC: "No heat tonight? We run 24/7."
- Electrician: "Power out? Don't wait — same-day service."

## Feature 2 — Service Area Section

**Priority**: v1  
**Applies to**: All trades  
**Blackboard field**: `service_area_cities` (string[], derived from GBP address + Agent 1 radius estimate)

### What it is

A section titled "Areas We Serve" (or "[City] & Surrounding Areas") listing the cities/counties the business covers. Displayed as either:
- A flex-wrap tag list (preferred for 5–15 cities)
- A 2-column text list (for 15+ cities)

### Why it matters

Contractors lose leads when searchers can't confirm the business serves their area. A visible service area list:
1. Confirms coverage for the visitor
2. Adds geographic SEO keywords (each city name is a ranking signal)
3. Reduces "do you come to [city]?" phone calls

### Data source

- Primary: GBP address city + county → Agent 1 infers a radius and generates a list of surrounding cities
- Fallback: If only one city is known, display just that city with "and surrounding areas"
- User override: Style Preferences step text field — "Any specific areas you serve?"

### Agent 1 prompt for service area inference

> "The business is located in [city, state]. It is a [category]. What cities and areas likely fall within a 15–25 mile service radius? List 8–12 city names, no county suffixes, comma-separated."

### DC/NoVA example output

`DC · Arlington · Alexandria · Fairfax · Reston · Herndon · McLean · Falls Church · Bethesda · Silver Spring`

## Feature 3 — License & Insurance Badge

**Priority**: v1  
**Applies to**: All licensed trades (plumbers, electricians, HVAC, roofers, general contractors)  
**Blackboard field**: `license_number` (string | null, from user input — not available from GBP)

### What it is

A trust strip or card section displaying:
- State contractor license number (e.g., "MD Lic. #54321")
- Insurance status badge ("Fully Insured")
- Optional: bonding status ("Bonded")

### Why it matters

License numbers are the #1 trust signal for contractors doing high-stakes work (gas lines, electrical panels, roofing). DC/NoVA homeowners specifically look for licensed contractors because of local permitting requirements.

Competitor sites almost never display license numbers prominently — this is a differentiator.

### Data input

GBP does not expose license numbers. Collected on the **Style Preferences** step (step 3) via optional input: "Do you have a contractor license number? (optional)"

- If provided → badge shows "MD/VA/DC Lic. #XXXXX · Insured"
- If blank → badge shows "Licensed & Insured" (generic, never AI-fabricated)

### Implementation notes

- Agent 6 renders a 3-chip trust row near the nav or hero: `[shield] Licensed` · `[check] Insured` · `[lock] Bonded`
- License number rendered in monospace for credibility
- Must not be AI-fabricated — show generic claim if no number provided

## Feature 4 — Google Reviews Badge

**Priority**: v1  
**Applies to**: All businesses with GBP reviews  
**Blackboard fields**: `google_rating` (float), `google_review_count` (int) — pulled live from Places API

### What it is

A compact badge in the hero showing the business's Google rating and review count, linking to the GBP listing.

Example: `★ 4.8 · 127 reviews on Google`

### Why it matters

Social proof is the fastest trust-builder for service contractors. A visible review count answers the homeowner's unspoken question: "Has anyone used this person before?"

### Data source

- Pulled from Google Places API response during the build flow (already on Blackboard as `google_rating` and `google_review_count`)
- Updated weekly via the `FEATURE_REVIEWS_BADGE` retention job (see `wiki/features/retention.md`)

### Implementation notes

- The deployment parser renders a stable `<a data-onara-review-badge>` block with rating + count text
- Star icons are pure CSS/SVG — no external icon dependency
- Falls back gracefully: if `google_review_count < 3`, badge is omitted entirely (avoid showing low counts)
- JSON-LD `AggregateRating` schema injected separately by Agent 8 (SEO)

## Feature 5 — Prominent Phone CTA

**Priority**: v1  
**Applies to**: All trades  
**Blackboard field**: `business_phone`

### What it is

Phone number displayed as a large `<a href="tel:...">` element in the hero — not buried in the footer. Tapping on mobile opens the dialer directly.

- Desktop: large phone number beside the main CTA button
- Mobile: sticky "Call Now" button fixed to the bottom of the screen (`position: fixed; bottom: 0`)

Agent 3 selects a high-contrast color for the phone block (safety orange on dark backgrounds, terracotta on light).

## Feature 6 — Services List

**Priority**: v1  
**Applies to**: All trades  
**Blackboard field**: `services` (string[], generated by Agent 2 from category)

### What it is

A card or list section titled "Our Services" showing the specific work the contractor does.

Example outputs by trade:
- **Plumber**: Drain cleaning · Water heater install · Re-pipe · Leak repair · Emergency plumbing
- **HVAC**: AC install · Furnace repair · Duct cleaning · Tune-ups · Emergency service
- **Electrician**: Panel upgrades · Outlet install · EV charger · Whole-home rewiring · Code compliance
- **Landscaper**: Lawn mowing · Mulching · Tree trimming · Irrigation · Seasonal cleanup

Agent 2 generates 5–8 items from the GBP category. User can append custom services via the Style Preferences step.

## Feature 7 — Free Estimate CTA + Contact Form

**Priority**: v1  
**Applies to**: All trades  
**Blackboard field**: `offers_free_estimate` (bool, default true for contractors)

### What it is

A secondary CTA alongside the phone number: "Get a Free Estimate →" linking to a simple contact form (name, phone, message).

### Why it matters

Some homeowners are not ready to call — they want to message first. A free estimate CTA captures this segment without requiring a booking system.

### Implementation

- Contact form POSTs to a Supabase Edge Function
- Edge Function sends a lead notification email when `FEATURE_LEAD_EMAIL` is on
- No third-party form service — Onara owns the lead flow

---

## Feature 8 — Years in Business Chip

**Priority**: v1 (if data available)  
**Applies to**: All trades  
**Blackboard field**: `founded_year` (int | null, from user input or GBP description)

### What it is

A small trust chip displayed near the business name or in the hero:
- "Est. 1998" or "Serving [city] since 2003"
- Displayed in monospace font as a credibility micro-element

### Data source

- Agent 1 extracts founding year from GBP business description if present
- Fallback: optional input on Style Preferences step ("What year did you start your business?")
- If unavailable: omitted; never AI-fabricated
